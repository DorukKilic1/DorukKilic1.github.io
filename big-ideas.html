Big Ideas Notes

These notes summarize key Java programming concepts and coding fundamentals.

  • Java: A programming language used worldwide to create software.
  • Algorithms: Step-by-step processes to complete tasks or solve problems.
  • Compiler: Error checker. Syntax errors are reported when found.
  • Logic errors: Most likely math errors.
  • Comments: // for single-line, /* */ for multi-line.
  • Variable: Memory location storing a value that can change during runtime.
  • Code examples:
    • System.out.println()
    • Double example: 6.353535
    • Int example: 4
    • Boolean: true or false
    • String: Words
  • To declare a variable: specify type, a space, the name, and end with ;.
  • Printing:
    • System.out.println(value) prints value + new line
    • System.out.print(value) prints value without new line
  • Data Types: Integers, doubles, booleans.
  • Documentation: Important to make code clear. Example: instruction manual, API.
  • Java Keywords:
    • public → accessible anywhere
    • class → blueprint for objects
    • static → belongs to class, not object
    • void → method that doesn’t return a value
    • main → starting point of program
  • API: Documentation that explains what a library can do and how to use it.
  • Incrementing a variable: a = a + 1;, a += 1;, a++;
  • User input: Scanner in = new Scanner(System.in);

    • Patterns: divisibility, min/max, digit finding, loops.
  • 2.10 String Algorithms
    • Replace chars in loop.
    • Traverse string with for-loop.
    • Count characters.
    • Reverse string.
    • Replace words.
  • 2.11 Nested Iteration
    • Nested loops = a loop inside another loop.
    • The inner loop must finish before the outer loop continues.
  • 2.12 Loop Analysis & Tracing
    • Statement execution count shows how many times a line runs.
    • Found by tracing loops step by step.
    • Trace tables track variable values each iteration.
    • Counting rules:
      • Simple counter loop → executions ≈ largest − smallest + 1
      • Nested loops → total ≈ outer loop × inner loop
  • 2.12.5 Loop Analysis Game
    • Practice tracing and counting loop executions.
    • Improves efficiency and accuracy.
  • 2.13 Selection (Conditionals)
    • Conditionals decide if code runs based on true/false.
    • Example: code runs only if x > 0.
    • Concepts:
      • Block of statements → { }
      • Boolean expression → true or false
      • Compound conditions → && and ||
      • Negation ! → flips true/false
    • Logic rules:
      • AND / OR follow truth rules.
      • De Morgan’s Laws apply negation to compound conditions.
      • Short-circuit evaluation stops early when result is known.
    • Java keywords:
      • if (condition) → runs when true
      • else → runs if if was false
      • else if (condition) → additional exclusive path
  • 2.14 Loops
    • Loops repeat code automatically.
    • Repeated code can be a single statement or { } block.
    • Loop types:
      • for loop → start, condition, update
      • while loop → runs while condition is true
      • Infinite loop → condition never becomes false
    • Important ideas:
      • Loop body may run zero or more times.
      • Out of bounds error → accessing past String or list length.
      • Tracing loops helps debug and understand behavior.
    • Java keywords:
      • while → creates a while loop
      • for → creates a for / for-each loop
      • System.out.println(variable) → trace and debug

Unit 3 — Abstraction and Object-Oriented Programming

  • 3.1 Abstraction and Program Design
    • Abstraction → show only what is needed.
    • Example: Scanner hides how input is read.
    • Identifying classes, attributes, and methods.
    • Nouns often become class names.
    • Top-down design (decomposition):
      • Break large problems into smaller methods.
    • Data abstraction:
      • Separates properties from implementation.
    • UML basics:
      • + public
      • - private
    • Instance variable → unique per object.
    • Static (class) variable → shared by all objects.
    • Why abstraction matters:
      • Reusability
      • Maintainability
  • 3.2 Impact of Program Design
    • Clear names make code easier to understand.
    • Organized code is easier to update and fix.
    • Generic methods and classes can be reused.
    • Small methods are easier to test.
  • 3.3 Anatomy of a Class
    • A class is a blueprint for creating objects.
    • A class typically includes:
      • Instance variables
      • Constructors
      • Methods
    • Instance variables are usually private.
    • Access is provided through public methods.
    • public → accessible outside the class.
    • private → accessible only inside the class.
  • 3.4 Constructors
    • Constructors initialize object state.
    • Rules:
      • Same name as the class.
      • No return type.
    • Default constructor exists if no constructor is written.
    • Overloading:
      • Multiple constructors with different parameters.
  • 3.5 Methods
    • Method structure:
      • accessModifier returnType methodName(parameters)
    • Return types:
      • void → no return value.
      • Non-void → must return a value.
    • Accessors (getters):
      • Return data without changing it.
    • Mutators (setters):
      • Modify object state.
    • Parameters allow generalization and reuse.
    • toString() returns a String representation of an object.
    • Preconditions → must be true before the call.
    • Postconditions → guaranteed after execution.
  • 3.6 Passing and Returning Objects
    • Objects are passed by reference copy.
    • Changes to a mutable object affect the original.
    • Defensive copying:
      • Prevents unintended changes.
    • Returning an object returns a reference, not a copy.
  • 3.7 Static
    • Belong to the class, not objects.
    • Called using ClassName.methodName().
    • Only one copy exists.
    • main method:
      • Always static.
      • One per class.
    • Static methods:
      • Cannot access instance variables unless given an object.
      • Can call other static methods directly.
    • Static variables:
      • Shared across all objects.
      • Useful for counters or shared values.
    • final keyword → makes a variable unchangeable.
  • 3.8 Scope
    • Scope → where a variable can be accessed.
    • Class scope → instance variables usable throughout the class.
    • Method scope → local variables exist only inside the method.
    • Block scope → variables inside loops or if-statements exist only there.
    • Local variables cannot be public or private.
    • Parameters are local variables.
    • Local variables can shadow instance variables.
    • Variables cannot be used outside their scope.
  • 3.9 this Keyword
    • Refers to the current object.
    • Used when parameters shadow instance variables.
    • Required to access instance variables in that case.
    • Not allowed in static methods.
